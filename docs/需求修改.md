### 1. `init` (初始化)
**核心职责：生成标准配置文件，降低上手门槛。**
* **行为**：
    * 在当前目录（或用户主目录 `~/.config/nanoskills/`）生成一个默认的 `.agent-skills.yaml` 文件。
    * 文件内应该包含默认的 `scan_paths`（比如 `.` 和 `~/scripts`）、`file_patterns` 以及带详细注释的示例。
* **边界处理**：如果文件已存在，切勿直接覆盖，提示用户“配置文件已存在”，或者提供 `-f, --force` 强制覆盖选项。

### 2. `sync` (同步/构建索引)
**核心职责：执行极速扫盘，刷新本地 JSON 缓存。**
* **行为**：
    * 读取配置文件中的路径和规则，利用 `ignore` 库并发扫描。
    * 调用你的“动态前缀解析引擎”提取所有合法的 Header。
    * 将结果序列化并覆盖写入到本地的缓存文件（例如 `~/.cache/nanoskills/index.json`）。
* **人类反馈**：执行完毕后，打印一条极其极客的耗时统计，例如：`✅ 扫描完毕！在 45ms 内遍历了 1200 个文件，成功索引 27 个技能。`

### 3. `list` (列出所有技能)
**核心职责：全局概览（类似于 `docker images`）。**
* **人类模式 (默认)**：读取缓存，使用 `comfy-table` 打印一个漂亮的 ASCII 表格，只展示最核心的三列：`名称`、`标签`、`描述`。
* **机器模式 (`nanoskills list --json`)**：直接吐出缓存里完整的 JSON 数组。Agent 启动时可以调用这个命令，一次性把所有工具挂载到自己的上下文中。

### 4. `pick` (交互式选择 - 你的绝杀武器)
**核心职责：人类用户的沉浸式模糊搜索。**
* **目前的痛点**：“只返回路径”在实际使用中有点鸡肋，因为用户选完通常是想**运行**它。
* **能力升级方案**：
    * **默认行为**：如果脚本有 `command_template`，返回**替换好 `{filepath}` 后的完整执行命令**（例如返回 `python /home/xxx/script.py`）。这样用户可以在外层包裹执行，比如 `eval $(nanoskills pick)`，选完直接运行！
    * **进阶选项**：提供 `--output` 参数。
        * `nanoskills pick --output path` (只输出路径，方便结合 `cat` 或 `vim` 用)。
        * `nanoskills pick --output json` (TUI 选完后，在终端吐出该技能的完整 JSON，方便管道传递给其他脚本)。

### 5. `search` (搜索技能)
**核心职责：Agent 的专属高频 API。**
* **参数要求**：需要接收一个 Query 参数，例如 `nanoskills search "图片格式转换"`。
* **能力**：读取缓存，计算匹配得分，过滤出结果。
* **输出区分**：
    * 不带 `--json` 时，打印类似 `list` 的表格，但只展示命中的。
    * **带 `--json` 时（极其关键）**：输出的 JSON 格式必须直接兼容 OpenAI 的 `tools` 数组规范（包含 `type: "function"`, `function: { name, description, parameters }`）。这是 `nanoskills` 征服大模型生态的核心卖点。

### 6. `info` (查看技能详情)
**核心职责：展示单体技能的微观参数。**
* **参数要求**：精确的技能名称，例如 `nanoskills info batch_file_organizer`。
* **能力**：
    * **人类模式**：在终端里把这个技能的 YAML 头部优雅地打印出来（如果你想炫技，可以在这里再次调用 `syntect` 把 YAML 高亮打印）。
    * **机器模式 (`--json`)**：只输出这一个技能的 JSON 对象。
